EchoVuln

A lab-focused vulnerability and posture tracking platform with an endpoint agent. It ingests host snapshots, normalizes findings, prioritizes risk, and tracks remediation status over time. Built to complement EchoSentinel (detection) and EchoAdversary (simulation) with posture lifecycle visibility.

Tooling Vulnerability Tracking Asset Inventory Risk Prioritization Remediation Lifecycle Agent-Based EchoPentest

Core Capabilities

  • Asset inventory with “last seen” tracking from the agent.
  • Vulnerability lifecycle: open → in progress → resolved (and reopen on re-detection).
  • Risk scoring based on severity + exposure + time-open (lab scoring model).
  • Evidence-first findings with remediation guidance fields.
  • Snapshot ingestion workflow to support repeatable posture checks.

Project Snapshot

Category
Tooling — Vulnerability Management / Posture
Architecture
Backend API + UI + Endpoint Agent
Inputs
Agent snapshots (host metadata + findings)
Outputs
Asset view, vuln list, severity filters, remediation tracking
Use Case
Labs and small environments (repeatable posture checks)

Agent Responsibilities

  • Collect host identity + baseline metadata
  • Generate a snapshot payload (JSON)
  • Submit snapshots to backend for normalization
  • Maintain stable agent identity (agent_id / tenant_id)

Workflow

1) Snapshot

Agent captures posture + findings and submits them as a structured payload.

  • Asset identification
  • Baseline metadata
  • Findings extraction (lab scope)

2) Normalize

Backend parses snapshots and converts them into a stable vulnerability model.

  • Deduplicate by host + finding
  • Apply severity
  • Update last seen

3) Track

UI shows posture over time and supports remediation lifecycle tracking.

  • Open / in progress / resolved
  • Time-open tracking
  • Risk-based prioritization
