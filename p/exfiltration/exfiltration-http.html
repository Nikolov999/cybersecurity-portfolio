Exfiltration over HTTP (CertUtil) Case Study

A defender-view validation of LOLBIN-driven HTTP transfer using certutil.exe. The flow starts from an established remote session, then pulls a payload over HTTP and writes it to disk. The evidence focuses on correlation across Sysmon Event ID 1 (process creation) and Windows Security 4688/4674 for process + privileged access context.

Purple Team Exfil / Transfer LOLBIN: certutil.exe HTTP (port 8000) Sysmon: Event ID 1 Security: 4688 Security: 4674 EchoSentinel MITRE ATT&CK

Key Findings

  • Execution: certutil.exe used with -urlcache -split -f to fetch a remote Base64 payload and write output to C:\Users\Public\out.b64.
  • High-signal telemetry: Sysmon clearly records the full command line for certutil.exe and the image path.
  • Process evidence: Security 4688 confirms process creation and the creator/subject context.
  • Privileged activity context: Security 4674 shows privileged object access attempts associated with the same execution burst.

Detection Design

Signal Chain (High Confidence)

Certutil is a legitimate Windows binary; confidence comes from the command line + network destination + write location.

  • Sysmon 1 — process creation with full CommandLine and image path
  • Security 4688 — additional process creation context (subject/creator)
  • Security 4674 — privileged object access attempts as enrichment
Correlation Logic EchoSentinel Queries

What To Alert On

  • certutil.exe with -urlcache + -split + -f
  • HTTP destination + uncommon port (example evidence uses :8000)
  • Output written to public/common locations (example: C:\Users\Public\out.b64)
  • Follow-on correlation via Process ID / Logon ID for attribution
