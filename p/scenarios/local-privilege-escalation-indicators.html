Local Privilege Escalation Indicators (Post-WinRM) Case Study

A behavior-focused continuation of the SMB credential abuse → WinRM lateral movement sequence. This phase validates how privilege escalation looks from the defender side: local admin group changes, elevated token issuance, and the immediate burst of privileged service/object activity that follows.

Purple Team Post-Lateral Movement Privilege Consolidation Local Administrators 4672 / 4673 / 4674 4732 EchoSentinel Windows Security Log MITRE ATT&CK

Key Findings

  • What changed: a remote session pivoted into durable admin control by modifying the local Administrators group (4732).
  • Proof of elevation: the next session received an elevated token (4672), confirming privilege materialization.
  • Behavioral confirmation: privileged service calls (4673) and privileged object access (4674) appeared immediately after elevation.
  • Analyst workflow: alert → pivot to raw event fields (Logon ID, subject, process) → correlate across the same session.

Detection Design

Signal Chain (High Confidence)

Each event type can occur during legitimate admin work. Confidence comes from chaining them after a remote session.

  • 4732 — local Administrators membership change
  • 4672 — elevated token issued after refresh
  • 4673 — privileged service calls (token actively used)
  • 4674 — privileged object access (protected objects touched)
Detection Pack Logon ID Correlation

Why This Works

  • Detects privilege consolidation without exploit signatures.
  • Anchors investigations on Logon ID to handle out-of-order timestamps.
  • Uses 4673/4674 as strong enrichment: “elevated token is being used.”
  • Remediation is measurable: group removal + recurrence alerting.
