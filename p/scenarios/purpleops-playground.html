PurpleOps Playground

An isolated attacker ↔ victim lab built to simulate a full single-host attack chain and document it as a defender play-by-play. Telemetry is captured through Wazuh alerts and Wireshark evidence, with reproducibility ensured via snapshots and an internal-only network.

Purple Team Blue Team Play-by-Play Wazuh Wireshark Internal Labnet MITRE ATT&CK Linux Telemetry Reverse Shell

Objectives

  • Run a controlled attack chain end-to-end (recon → initial access → execution → persistence → privilege escalation).
  • Capture high-signal defender artifacts (alerts + network traces) and document detection gaps.
  • Keep the lab isolated and repeatable with snapshots and an internal network.

Attack Chain

Reconnaissance

Active scanning against the victim to identify reachable services and confirm target responsiveness in the internal labnet.

  • MITRE: T1595
  • Evidence: scanning artifacts in Wazuh and Wireshark

Initial Access

Payload retrieval over HTTP and operator-driven execution flow, captured as network and host artifacts.

  • MITRE: T1204 (User Execution)
  • Evidence: file download observed in Wireshark

Execution

Remote session established, followed by observable malicious behaviors including brute-force and injection attempts.

  • MITRE: T1059
  • Evidence: Wazuh alerts + network traces

Persistence

Backdoor user creation and privilege modifications designed to survive session loss and enable repeated access.

  • MITRE: T1136
  • Evidence: user creation, sudoers modifications, password changes

Privilege Escalation

Escalation path validated with clear post-escalation confirmation. Telemetry focused on suspicious sudo flows.

  • MITRE: T1548
  • Evidence: escalation alerting and traceability

Detection Gaps

Notes on what was not visible by default, where correlation was required, and what additional instrumentation would improve coverage.

  • Example: file-creation logging gaps in /tmp flows
  • Action: add stronger host instrumentation where required
