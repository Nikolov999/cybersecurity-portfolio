DNS Beaconing & Suspicious Resolution Analysis

Controlled DNS beacon simulation performed against echopentest.com using repeated resolution cycles. Telemetry collected via Microsoft-Windows-DNS-Client/Operational and ingested into EchoSentinel. Detection engineered around frequency anomalies and repetitive query patterns.

Purple Team DNS C2 Simulation Event IDs 3006/3008/3010/3018 EchoSentinel Frequency Analysis Detection Engineering MITRE T1071.004
Project README Detection Analysis Lab Commands

Detection Logic

  • Repeated DNS queries at 5–10 second intervals
  • Single domain repeatedly resolved
  • Consistent DNS server (8.8.8.8)
  • Mixed success / failure events
  • Stable periodic timestamps

This behavior strongly resembles DNS-based command-and-control beaconing, where malware maintains periodic communication through DNS resolution.
