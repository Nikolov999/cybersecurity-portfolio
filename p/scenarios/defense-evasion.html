Defense Evasion & Log Tampering Case Study

A defender-view validation of coordinated security impairment following WinRM remote access. The evidence focuses on correlating authentication context with subsequent changes that reduce visibility: Microsoft Defender tampering, audit policy modification, and Sysmon driver unload. Supporting artifacts include an EchoSentinel hunt JSON capturing the full event set.

Purple Team Defense Evasion WinRM Security: 4624 Security: 4672 Defender: 5007 Defender: 5001 Security: 4719 System: 1 Rule: ES-DE-001 EchoSentinel MITRE ATT&CK

Key Findings

  • Remote access anchor: Security 4624 with network logon context followed immediately by 4672 privileged token assignment.
  • Defender impairment: Defender operational events show a configuration change (5007) and a protection state change (5001).
  • Audit reduction: Security 4719 indicates system audit policy modification (visibility downgrade risk).
  • Telemetry degradation: System event 1 captures Sysmon driver unload (coverage loss during the execution window).
  • Behavioral correlation: ES-DE-001 validates that sequencing and proximity across these events is higher-signal than any single event.

Detection Design

Signal Chain (High Confidence)

Confidence comes from temporal proximity and intent inference: authenticated privileged access followed by multiple independent visibility-reduction indicators.

  • Security 4624 — remote logon anchor (network logon)
  • Security 4672 — privileged token assignment (admin context)
  • Defender 5007/5001 — tamper/state change indicators
  • Security 4719 — audit policy modifications
  • System 1 — Sysmon driver unload (coverage loss)
ES-DE-001 Logic Hunt JSON

What To Alert On

  • Privileged remote session context (4624 + 4672) followed by impairment signals inside a tight time window
  • Defender configuration or state changes (5007, 5001) that reduce protections or alter enforcement posture
  • Audit policy changes (4719) that affect event capture fidelity
  • Telemetry degradation (System 1 SysmonDrv unload) with no approved change record
