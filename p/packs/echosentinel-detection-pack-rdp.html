Rules Created (EchoSentinel)

ES-AUTH-001 — Brute Force Suspected

4625 threshold for same hostname + username + source_ip within brute_fail_window_seconds. Default: 5 failures within 120 seconds.

MITRE: T1110 • Severity: Medium

ES-AUTH-002 — Password Spray Suspected

4625 from same source_ip with high failure count and multiple distinct usernames within a time window (spray model).

MITRE: T1110 • Severity: High

ES-AUTH-003 — RDP Brute Force (LogonType 10)

4625 where LogonType = 10, same source_ip + username, threshold 3 failures within 60 seconds. Primary RDP brute detection.

MITRE: T1110 • Severity: High

ES-AUTH-004/005/006 — Anomaly + Privileged Monitoring

Rapid IP change for user (4624), privileged user logons (admin list), and first-seen source IP for user via baseline persistence.

Tuning, Suppression, and False Positives

Threshold Tuning Applied

  • brute_fail_window_seconds reduced from 120 → 90
  • spray_fail_threshold increased from 15 → 20
  • RDP brute defaults retained: 3 failures / 60 seconds

Goal: reduce noise from password retries while keeping brute sensitivity.

Observed False Positives

  • Admin mistyping password → ES-AUTH-001
  • Legit admin RDP from new workstation → ES-AUTH-006
  • Maintenance credential testing → ES-AUTH-002

Mitigations: admin allowlist, spray threshold increase, baseline suppression, severity-based suppression windows.
