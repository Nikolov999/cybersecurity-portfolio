EchoSentinel Detection Pack — WinRM & Lateral Movement

Reusable detection logic for WinRM sessions: WinRM Operational evidence, Security authentication context, provider-host lineage (wsmprovhost.exe), and content-level proof via PowerShell logging.

Listed Queries Rules WinRM T1021.006

Telemetry Sources

  • Microsoft-Windows-WinRM/Operational: session activity evidence
  • Security: 4624, 4625, 4648, 4688 (if enabled)
  • PowerShell Operational: 4104 (script block), 4103 (module)
  • Sysmon (optional): EID 1/3 for process + network tie-in

High-Signal Anchors

Session
WinRM Operational events in window + matching source IP
Auth
Security 4624 (often LogonType 3) with remote source
Lineage
svchost (WinRM) → wsmprovhost.exe → shell
Content
PowerShell 4104 containing executed strings/markers

Process Chain Expectations

Canonical Chain

svchost.exe (WinRM service) → wsmprovhost.exe (provider host) → child process (powershell.exe / cmd.exe) with optional conhost.exe.

Anchor pivot: presence of wsmprovhost.exe during the activity window.

Content-Level Indicators

PowerShell script blocks containing Invoke-Command, Enter-PSSession, New-PSSession and download/execute primitives (risk escalation).

Correlation Logic

Correlation 1 — WinRM Session + Logon

WinRM Operational activity + Security 4624 in the same window, with matching source IP. Output: confirmed WinRM auth/session.

Correlation 2 — Logon + Process Chain

4624 followed by process creation evidence of wsmprovhost.exe and its children. Output: remote session executed commands (provider-host lineage proof).

Correlation 3 — Script Block Confirmation

PowerShell 4104 shows executed content (whoami/hostname/unique marker). Output: remote execution with content-level proof.

Severity Guidance

High if source IP is not a known management host, or user is privileged. Critical if script block includes download/execute primitives.

Validation (Expected Alerts)

Alert Titles

  • WINRM Remote Session (Non-Admin Host)
  • WINRM Provider Host Spawned Shell
  • PowerShell Remoting Script Block (WinRM)
  • Blocked WinRM Attempt (Non-Allowlisted Source) (optional)

Test Cases Covered

  • Successful WinRM session + whoami
  • RCE burst (multiple commands)
  • Failed WinRM after hardening
  • Allowlisted management access
  • 4104 marker string validation
