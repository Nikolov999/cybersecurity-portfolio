EchoSentinel Detection Pack — Remote Service Creation & Persistence

Detection engineering for service creation and persistence using correlated telemetry: sc.exe create execution, service install confirmation, registry ImagePath writes, payload staging under ProgramData, and execution lineage.

Listed Queries Rules Remote Service Creation T1021.006

High-Signal Detections

  • Sysmon 1: sc.exe with create (binPath evidence when present)
  • Service install event correlated to sc.exe create
  • Sysmon 13: HKLM\System\CurrentControlSet\Services\ ImagePath set
  • Sysmon 11: ProgramData payload drop + Sysmon 1 execute

Extraction Anchors

Creator
Sysmon 1 Image/CommandLine for sc.exe create
Install
Security “A service was installed” + time correlation
Persistence
Registry ImagePath write (Sysmon 13)
Execution
Payload executed from ProgramData + child spawn

Detection Logic

1) sc.exe creating a service

Sysmon EID 1 where Image endswith \sc.exe and CommandLine contains create. Optional boost if CommandLine contains binPath=.

2) Service install + sc.exe correlation

Correlate Sysmon 1 sc.exe create with service installation event near the same time. Output: explainable and high confidence.

3) ImagePath registry set

Sysmon EID 13 where TargetObject contains \System\CurrentControlSet\Services\ and \ImagePath.

4) ProgramData payload drop + execute

Sysmon EID 11 under C:\ProgramData\, followed by Sysmon EID 1 executing that path. Add scoring if payload spawns cmd/powershell/whoami or uses output redirection.

Queries (Manual Hunt)

sc.exe service creation

channel: Microsoft-Windows-Sysmon/Operational
event_id: 1
image endswith: \sc.exe
command_line contains: " create "

Service ImagePath registry edits

channel: Microsoft-Windows-Sysmon/Operational
event_id: 13
target_object contains: "\System\CurrentControlSet\Services\"
target_object contains: "\ImagePath"

Payload staged in ProgramData

channel: Microsoft-Windows-Sysmon/Operational
event_id: 11
target_filename startswith: "C:\ProgramData\"

Validation Pass Criteria

Command lines, parent/child, file paths, registry target objects visible. Alerts trigger on correlations, not single noisy events.
